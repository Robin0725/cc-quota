# CC Quota 错题集

排障前先查同类案例;修完必记:症状 → 真凶 → 修法 → 教训。

## #001 面板内容无声消失(×2)

- **症状**:面板卡片内容整块不见,无报错。
- **真凶**:一次是 `slice(0, 2)` 截断;一次是 CSS 行高压缩 + `overflow: hidden` 裁掉 Claude 的 Fable 分桶行。
- **修法**:`grid-auto-rows` 保 `min-content` 下限;不加 `overflow: hidden`。
- **教训**:这两道安全网是防静默丢数据的,改面板布局不得拆。

## #002 Kimi 卡片每刻钟整个消失

- **症状**:Kimi CLI 闲置时,菜单栏 Kimi 胶囊每 ~15 分钟消失一次。
- **真凶**:`merge_snapshots` 按 status 把 `signed_out` 当"永久失败"清空读数;而 Kimi token 只活 15 分钟,闲置过期是常态。
- **修法**:只有 `ok` 才替换,其余一切失败退回 30 分钟内上次读数并标 `stale`(合同附录)。
- **教训**:不得按 status 猜"失败是否永久";保留上次读数由 `MAX_STALE_SECONDS` 界定。

## #003 悬浮窗被后台 agent 劫持,固定显示 kimi(2026-07-19,v0.5.2 修)

- **症状**:用户在 Claude 里打字,悬浮球却固定显示 kimi 不切换。
- **真凶**:活动信号用"session 目录最近写入"。一个后台跑长任务的 Kimi agent 每一两分钟流式写盘,"最近写盘者"几乎永远是它。信号测的是 **agent 在干活**,不是**用户在用谁**。
- **修法**:信号改为只有用户亲自输入才变动的提示历史(claude → `~/.claude/history.jsonl`,kimi → `~/.kimi-code/user-history/`);codex 无等价文件,留 sessions,已知局限(合同 §6.1b)。
- **教训**:选"用户正在用谁"的信号时,先问"后台 agent 会不会也触发它"。凡 agent 也会写的路径都不合格。
- **附带发现**:当时线上跑的是 03:53 的 debug 构建(`/Applications` 里也是 debug 包),不含后续修复——发版后要确认用户机器上换成了 release 产物。

## #004 全仓评审三连(2026-07-19,v0.5.2 修)

1. 前端 backoff 把"任一 provider 非 ok"计为失败 → Kimi 闲置(常态)导致轮询钉死在 30 分钟顶格、临近重置的 60s 快路径永不生效。改为"有人成功即健康"。
2. keychain 存在性探测(`security` 子进程)无超时、在 750ms 轮询链路上、不受 PROVIDER_FETCH_BUDGET 覆盖 → 锁定的 keychain 可无限挂起。加 2 秒上限,超时杀子进程按"不存在"处理。
3. 冷启动窗口(取数进行中 + 缓存为空)碰撞请求用 `providers::all()` 造出未登录 provider 的幻影卡。改 `active()`。
